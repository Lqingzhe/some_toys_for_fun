# 功能介绍  

### 前言
你还在为忘记定期向女朋友发送早安、晚安吗？你还在担心不能每天关心父母吗？  
这个以LLBot为核心，用go语言编写的小玩意或许能帮到你。 

### 用处
这个仓库可以让你能够每天，准时、准点、风雨无阻、一天不落地在qq中发送消息，你只需要提前设置好发送的时间，以及内容（可以多条内容随机发送一条），之后每天重复发送的事情就可以交给它处理。~~解放双手~~ 显示自己的对他、她或他们的关心。

# 开始使用

### **补充一点，该项目使用了docker容器化技术，不想使用docker的可以本地部署[LLBot](https://luckylillia.com/)**

## 配置文件
以下的时间为24小时制，hour和minute不填默认为0，**强烈建议消息的顺序按照时间顺序，不然会出现某些消息不发送的情况**   
ps:作者懒得加排序了，写配置的时候注意下 --~~顺手的事情~~
```yaml
global:
  path: "http://llonebot:3000/" #使用docker的朋友地址就如果不改动docker相关文件就无需改动
                                #本地部署的朋友使用  http://localpost:消息监听的端口号 
                                #默认是3000，具体的其实我也没试过本地部署，点击上面的链接看看LLBot的作者怎么说 :D

tasks:
  - qq_id: "”         # 输入接收者的qq号或者群的群号
    private: true     # 如果目标是用户填写true，如果是发给群里面就是false
    info:             # 这行不用管，下面的每一条例如例子中的 morning,noon 等只是一个名字，没啥用处，同一个目标用户不重复就ok
      morning:        # 一个名字，表示这是一条消息，没啥实际用处
        hour: 7       # 发送消息的时间的小时
        minute: 0     # 发送消息时间的分
        message:      # 后面填写你想发送的消息内容,填写多条就是随机一条发送
          - "ping 1"  # 第一条消息中可能发送的内容
          - "ping 2"  # 第一条消息中可能发送的内容
      noon:           # 第二个名字，表示这是第二条消息，没啥用处
        hour: 12      # 发送消息的时间的小时
        minute: 0     # 发送消息时间的分
        message:      # 后面填写你想发送的消息内容,填写多条就是随机一条发送
          - "ping 3"  # 第二条消息的内容（因为只有这一条）
  - qq_id: ""         
    private:
    info:
      name1:
        hour:
        minute:
        message:
          - ""
      name2:
        hour:
        minute:
        message:
          - ""
          - ""
```
## 启动项目

### docker部署
我会给你最简单，最直白，最有效的方式，一句话结论：执行 `docker compose up -d --build `，然后等待容器构建完成。跳过后面的本地部署部分，接着往下看。  
ps:如果提示LLBot的镜像拉不下来，记得打开你的梯子，中国国内的镜像源大概率用不了，实在不行问问ai，多试试（别问，问就是作者有故事）

### LLBot基本配置
1. 浏览器访问配置文件中的地址的3080端口（如果改了docker-compose配置的这里跟着改）
2. 随便设置一个密码，别忘了就行
3. 使用要发送的qq进行扫码登陆
4. 点击左侧`OneBot11`，点击http服务，启用此适配器，端口填写`3000`，Token填写`114514`(作者懒得改了，后面有改Token的方法)，然后保存。

#### 改Token的方法
找到文件./api/api.go中的SengMessage方法（函数）的`req.Header.Set("Authorization", "Bearer 114514")`这一行，把`114514`改成自己的Token就行
